2025-10-02 (THURSDAY): PHISHING PAGES USING BLOB URLS

AUTHORS:

- Peng Peng, Alex Starov, Beliz Kaleli, Lucas Hu, Shawn Shuang

NOTES: 

- We observed several active phishing campaigns using blob URLs to deliver the final phishing pages. 
- Blob pages dynamically construct malicious content in a browser's memory at runtime to bypass network analyzers.

DETAILS:

- A blob URL is a temporary web address that references data stored in the browserâ€™s memory.
- This allows developers to access and manipulate data on a client without requesting the data from a server.
- Blob URLs typically appear in a browser's navigation bar, starting with "blob:" and ending with a UUID.
- These UUID values in blob URLs are different for each attempt at accessing the parent URL.
- Blob URLs use the following format:

  blob:https://maliciousdomain[.]tld/01234567-89ab-cdef-0123-456789abcded

- Phishing sites can implement blob URLs through base64 text that represents the final phishing page. 
- Upon site access, the base64 text is decoded and assembled into a blob object within the victim's browser.

- For example, the following URL:
  -- hxxps[:]//trungtamdulieu.khaihoanprime[.]vn/wp-admin/js/cgi-/trade/blob.html
- generated this blob URL in the web browser when we checked on 2025-10-02:
  -- hxxps[:]//trungtamdulieu.khaihoanprime[.]vn/d9d26e2c-fac8-41eb-8e7d-2630b138a5cd

- Some URL chains from this activity leverage meta refresh for malicious redirection.

- For example, the following URL:
  -- hxxps[:]//abctraveladvisor[.]com/wp-admin/css/red/wx.htm returned the following HTML text:
- Returned the following text (defanged here) as recently as 2025-10-02:
  <META HTTP-EQUIV="refresh" CONTENT="0.1;URL=hxxps[:]//trungtamdulieu.khaihoanprime[.]vn/wp-admin/js/cgi-/trade/blob.html">

- Some URLs from this activity abuse 3rd-party storage services to host the malicious content.
- A chain of events active as of 2025-10-02 using a 3rd-party storage service for a blob URL follows:
  -- hxxps[:]//curly-sky-464f.ivelissayed74236719.workers[.]dev
  -- hxxps[:]//firebasestorage.googleapis[.]com/v0/b/my-office-documents.appspot[.]com/o/my-oned2023.html?
     alt=media&token=c894f398-084a-46d2-ab5f-e1d5457fe3f7
  -- blob:hxxps[:]//firebasestorage.googleapis[.]com/c894f398-084a-46d2-ab5f-e1d5457fe3f7

- We have seen blob URL-based phishing used to mimic a variety of companies and organizations.
- These may mimic a company's login page, or they could mimic web-based services, like email providers.
