2026-06-16 (TUESDAY): TITAN INFOSTEALER EMBEDDED IN AI ASSISTANT PYPI PACKAGE

AUTHORS:

- Mark Lim, Jingwen Shi, Tyler Halfpop, Ashutosh Chitwadgi

NOTES:

- Disguised as an AI assistant, TITAN (formally "TITAN Advanced Tools v3.0") is a Python-based infostealer.
  -- This infostealer employs Optical Character Recognition (OCR) for continuous screen monitoring.
  -- It exfiltrates captured screenshots to an attacker-managed Gemini API account. 
- Exfiltration uses HTTPS traffic to third-party cloud infrastructure at generativelanguage.googleapis[.]com. 
  -- Consequently, there are no custom C2 domains to block.
  -- The activity appears identical to legitimate Gemini API developer traffic. 
- A daemon thread polls the screen every 8 seconds and delivers social engineering prompts.
  -- These manipulate a victim into actively triggering screen capture at high-value moments.
  —- These screen captures include email, login forms, IDE windows, and checkout pages.
- TITAN is embedded as aria_modules/aria_advanced_tools.py within the PyPI package myra-ai-assistant.
- TITAM was published by an account created on 2026-05-31 with no prior publication history.
- Another module (aria_login.py) silently exfiltrates all myra-ai-assistant user conversation history.
  -- This is delivered to an attacker-controlled Firebase project via hardcoded credentials.
- The package's face authentication module (face_auth.py) silently exfiltrates biometric enrollment photos/
  -- These go to the same Firebase project upon first user login
  -- The module exfiltrates face data as a secondary PII channel.
- An earlier version of the same module set (aria_modules v0.1.0) contains identical Firebase project credentials.
  -- This confirms the exfiltration infrastructure predates the PyPI package
  -- This represents an active multi-version development effort
  -- The Firebase admin SDK service account key was rotated between v0.1.0 and v0.2.3.
- VirusTotal detections: 0/63 as of 2026-06-09

DETAILS:

Phase 1: Silent Deployment (aria_advanced_tools.py)

- Installs the open source Tesseract OCR silently via winget or Chocolatey at import time 
- Gemini API key accepted at runtime via init_advanced() 
— No hardcoded credential to detect statically

Phase 2: Proactive Screen Surveillance (aria_advanced_tools.py)

- _proactive_loop() detects screen state changes via MD5 thumbnail hash
- _suggest_from_context() matches the active window title against high-value categories:
  —- login/signup/checkout forms
  -- error/crash screens
  -- IDE windows (VS Code, PyCharm)
  -- email clients (Gmail, Outlook)
  -- spreadsheets
  -- PDF files
  -- WhatsApp
— _suggest_from_context() also delivers a category-specific Hindi-English (Hinglish) social engineering prompt through the AI persona
  -- This attempts to make victims active and willing participants in their own screen capture

Phase 3: OCR + Gemini Exfiltration (aria_advanced_tools.py)

- Full screenshot and three-strategy Tesseract OCR text are sent to the attacker's Gemini account
  -- This is done via generativelanguage.googleapis[.]com
— All exfiltration routes through third-party cloud infrastructure with no custom C2 endpoint

Phase 4: Persistence (aria_advanced_tools.py)

- Reboot-persistent Windows scheduled task is created
  -- This is done via PowerShell Register-ScheduledTask using a Toast notification payload as cover
  -- Sessions are logged to ~/Documents/TITAN_Data/memory/TITAN_memory.db

Phase 5: Package-Level Firebase Exfiltration (aria_login.py, face_auth.py)

- aria_login.py exfiltrates every conversation turn to Firebase users/{uid}/messages/{msg_id}
- face_auth.py silently uploads biometric enrollment photos to TITAN_faces/{os_user}_{hostname} 
  -- This is done in a background thread on first login.
