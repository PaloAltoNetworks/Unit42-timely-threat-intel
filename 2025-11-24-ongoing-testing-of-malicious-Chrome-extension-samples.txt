NOTES:

- We're tracking a threat actor who is testing samples of a malicious Chrome extension on VirusTotal.
- This is likely done to evade detection before deployment. 
- These malicious samples impersonate the legitimate Norton Safe Search Enhanced extension.
- The malicious samples use obfuscated JavaScript (JS) for communication with command and control (C2) servers.

DETAILS:

- The legitimate extension being impersonated is Norton Safe Search Enhanced
  -- The legitimate Norton extension ID is eoigllimhcllmhedfbmahegmoakcdakd
  -- It currently reports 100k+ users.
- The malicious samples use the same icons, name and description as the legitimate extension. 
- All of the malicious JS code in this extension resides in a single file.
  -- Located within the extension package at: /js/background.js
  -- The JS code in this file is highly obfuscated.

GROUPS AND TIMELINE OF THE SAMPLES:

- So far, we've identified at least 395 samples submitted to VirusTotal for this extension since 2024-08-18.
  -- 393 of these used the same VirusTotal API source identifier (223274c1).
  -- These were submitted from the same location, suggesting they belong to a single submitter. 
- The samples were submitted IOC using 6 different names:
  -- cworld.crx (102 submissions)
  -- cworld.crx0 (34 submissions)
  -- dsuc.crx (123 submissions)
  -- dsue.crx (86 submissions)
  -- norton.crx (39 submissions)
  -- norton.crx0 (11 submissions)
- New samples are still appearing in VirusTotal as we write this.

VARIATIONS OF THE SAMPLES:

- The threat actor re-packed content of the samples for each submission.
- This means each submission has a different SHA256 hash.
- Via a code comparison using diff, we found 79 variations of the JS code.

PRELIMINARY ANALYSIS OF THE EXTENSIONS:

- Since the JS code is highly obfuscated, we reviewed its activity through dynamic analysis.
  -- The extension uses the chrome.alarms API to schedule part of the JS code to run periodically.
     --- The same patterns and specific periods are shared among the versions as follows:
         ---- periodInMinutes = 720 : cworld.crx, cworld.crx0, norton, notron.crx0
         ---- periodInMinutes = 363 : dsue.crx
         ---- periodInMinutes = 330 : dsuc.crx
- C (C2) traffic consists of GET requests over HTTPS to attacker-controlled servers.
- This malicious extension reuses part of the official Norton extension's implementation. 
  -- This tactic is likely used to make samples of this malicious extension appear more legitimate.

LINKS TO OTHER MALICIOUS EXTENSIONS:

- Other malicious extensions generate C2 traffic with URLs similar to C2 URLs for this extension.
  -- One example is a malicious extension from earlier this year that impersonated Snap Color Picker
     --- Extension ID: nbpljhppefmpifoffhhmllmacfdckokh

TAKEAWAYS:

- We assess that these samples have been submitted to VirusTotal as a development tactic.
- Attackers behind this malicious extension are likely adjusting and testing samples to better evade detection.
- If these samples can evade detection in VirusTotal, they will likely evade detection by Chrome and Google.
- So far, we haven't observed this extension on the Chrome Web Store.
- We found evidence from 2024 that a version of this extension was installed by a Windows malware EXE file.
- However, we have not yet seen current versions of this extension deployed for an attack. 
- Analyzing this type of data can reveal new families of malicious Chrome extensions.
