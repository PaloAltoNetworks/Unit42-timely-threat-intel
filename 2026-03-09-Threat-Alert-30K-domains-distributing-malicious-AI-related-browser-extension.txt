2026-03-09 (MONDAY): Threat Alert: 30K+ Domains Distributing Malicious AI-Related Browser Extension

AUTHORS:

- Qinge Xie, Shehroze Farooqi, Nabeel Mohamed, Shresta Bellary Seetharam, Fang Liu, Gaurav Rane

NOTES:

- Campaign distributes extension:
-- We discovered 30K+ domains that perform multiple redirects before they eventually route traffic to a landing page used for distributing a browser extension named "OmniBar AI Chat and Search". 
-- The final landing web page is hosted at the malicious IOC ansiblealgorithm[.]com. 
-- We observe the utm_campaign parameter is present in the final web page URL (example URL:hxxps[:]//ansiblealgorithm[.]com/new/?a=static-clean-aisrch&utm_campaign=69948c5fd82a7bd9c6520c49), which suggests this activity is likely part of a large-scale affiliate marketing scheme. 

- The "OmniBar AI extension" behavior: 
-- HomePage Override: The extension overrides Chrome settings (via chrome_settings_overrides API) to force the browser homepage to omnibar[.]ai and sets the default search provider to a custom URL (go.omnibar[.]ai/?api=omni&sub1=omnibar.ai&q={searchTerms}​​) that tracks queries via an API parameter.
-- Search Interception: The background.js file in the extension utilizes the chrome.search API to intercept user search inputs. It redirects the browser to a developer-controlled endpoint (hxxps[:]//go.omnibar[.]ai/?api=ai&q={searchTerms}), allowing the threat actor to monetize traffic and log sensitive data.

- The campaign is scaling: 
-- Following the discovery of the "OmniBar AI Chat and Search" extension, our engine later observed a second extension, "AI Output Algo Tool," being distributed via the same campaign. This extension exhibits browser-hijacking behavior similar to the OmniBar extension, including homepage override and search interception. Chrome Web Store data shows the user base doubled from 1K users on February 18, 2026, to 2K users by March 2026. This rapid increase suggests the campaign is effectively luring more users into installing these extensions.

- Deceptive legitimacy & attacker-controlled domain: 
-- The extension’s behavior matches its Web Store description. By overriding settings to provide "AI features," it appears to be "working as intended" to the user, effectively minimizing suspicion and bypassing traditional behavioral detections. We identified this campaign as our engine correlated the extension’s distribution layer with a known malicious IOC. 
-- Our testing observed that the developer-controlled endpoint is redirected to legitimate search engines and Gen AI services to build trust while maintaining control. 
-- Although the search API currently points to legitimate services to build trust, the endpoint is controlled by the attacker. By routing traffic through their own API first (rather than pointing directly to the widely known search engines and Gen AI services), the attacker maintains a "remote switch." This allows them to instantly pivot the redirection to malicious payloads or phishing sites at any time, without updating the extension’s code or alerting the user.

- High-risk extensions from the same developer: ​​Given that the campaign is actively distributing multiple extensions from its developer, we conducted a broader investigation into other extensions by the same author. We identified additional high-risk extensions from the developer, including: 
-- Search interception: We identify that the "serpey.com official extension" exhibits browser-hijacking behavior similar to the OmniBar extension, including home page override and search interception. Its Chrome Web Store description also mentions OmniBar search: "This simple extension alters your default search engine so that omni bar searches use serpey[.]com."
-- Affiliate adware: Extension "Care.Sale" and "giant Coupons Official Extension" track user browsing activity to inject affiliate markers and redirect traffic. 
-- Data exfiltration: Extension "Consensus - Reddit Comment Summarizer" extracts and transmits user Reddit comment threads to a developer-controlled API endpoint.
