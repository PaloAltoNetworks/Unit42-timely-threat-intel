2026-05-15 (FRIDAY): Malicious ads targeting LinkedIn leads to CastleLoader delivering AsyncRat

AUTHOR(S):

Thomas Story, Jonny Westfall

NOTES:
Phase 1:
- User executes obfuscated PowerShell commands via "Clickfix" lure.
-- Displays fake LinkedIn verification while downloading script. 

Phase 2: CastleLoader & Python Dropper 
- NSIS installer sideloads Python 3.12. 
- Malicious Python script injects encrypted shellcode directly into memory. 
Phase 3: Downloader Shellcode • Fetches final stage from C2 using spoofed Chrome headers. 
- Uses `ReplaceTextW` callback technique to evade detection. 
Phase 4: Final Payload 
- Reflective PE loader unpacks and runs AsyncRAT in memory. 

DETAILS:
Phase 1: The Lure & Initial Execution (Clickfix)
- The attack begins with a "Clickfix" lure prompting the user to run malicious powershell commands.
-- PowerShell Obfuscation: The initial command uses string manipulation ($WarningPreference) to hide the irm (Invoke-RestMethod) command.
-- Deceptive UI: It downloads and executes a script from ookinago[.]com/manera.txt, which displays a fake "Verification complete!" message box labeled "linkedin.com" to lower the user's suspicion while the infection proceeds in the background.
Phase 2: CastleLoader & Python Dropper
- The PowerShell script downloads an NSIS (Nullsoft) installer, which acts as CastleLoader.
-- Sideloading: The installer bundles a legitimate Python 3.12 runtime.
-- Encrypted Script: It executes a malicious Python script (pantography.txt).
-- Shellcode Injection: This Python script contains a Base64-encoded, XOR-encrypted block of shellcode. It uses the ctypes library to allocate memory in the heap and move the shellcode into memory for execution, bypassing traditional file-on-disk detection.
Phase 3: The Downloader Shellcode
- Once the Python script triggers the shellcode, a sophisticated downloader takes over:
-- C2 Communication: It dynamically constructs a URL on the stack (guckevichrealestate[.]com/.../v6) to fetch the next stage.
-- Evasion: It uses WinINet APIs with a spoofed Chrome User-Agent to blend in with normal web traffic.
-- Advanced Execution: To run the downloaded payload, it uses a rare callback technique (ReplaceTextW from comdlg32.dll), which helps evade security products monitoring for standard process injection.
Phase 4: Final Payload (AsyncRAT)
- The final stage is a Reflective PE (Portable Executable) loader that unpacks and executes the AsyncRAT malware in memory.
